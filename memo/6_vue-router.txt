▷ SPA (Single Page Application) > 인스타 생각하기!
	단일 페이지로 구성된 웹 애플리케이션
	SPA는 처음에 웹 애플리케이션의 모든 컨텐츠를 한 번에 로드하고, 이 후 페이지 간의 이동이나 컨텐츠 업데이트가 필요할 때마다 서버 전체 페이지를
	다시 요청하는 대신 JS를 사용하여 필요한 부분만 동적으로 로드하거나 업데이트 하는 방식

▶ SPA 장점
	1) 빠른 페이지 로드
		초기 로드 시 전체 애플리케이션을 불러오므로 이후 페이지 전환 시 매우 빠른 응답 속도를 제공한다.

	2) 효율적인 데이터 처리
		서버와의 통신은 필요한 데이터만 주고 받으므로 네트워크 트래픽이 줄어들고 성능이 향상 된다.

	3) 사용자 경험(UX)의 차이
		페이지 전환 시 전체 페이지를 다시 로드하지 않으므로 사용자는 자연스럽고 부드러운 사용자 경험을 얻는다.

▶ vue-router
	vue.js의 공식 라우터 라이브러리, SPA를 손쉽게 만들 수 있도록 여러 기능을 제공한다.

	기본 문법
	<template>
		<router-link class="card" to="/">홈</router-link>
		<router-link to="/free">자유게시판</router-link>
		<router-link to="/anony">익명게시판</router-link>
	
		<router-view></router-view>
	</template>

	라우터
	routes: [
		{ path: '/', component: Home.vue(컴포넌트 경로) }
		{ path: '/free', component: Free.vue(컴포넌트 경로) }
		{ path: '/anony', component: Anony.vue(컴포넌트 경로) }
	]

▶ 폴더 구성
	components: 재사용 가능한 작은 컴포넌트들을 포함
	pages: 라우트에 매핑되는 페이지들을 포함
